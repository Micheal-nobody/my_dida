# 新版本更新摘要

## 新增功能

## 优化与改进

- **在任务卡片最左侧显示所属清单颜色竖条**：优化了 `TaskCard` 组件，在任务卡片的最左侧添加了能够指示其所属清单颜色的 4px 宽圆角裁剪竖条，并对主任务列表页、看板视图、日历任务列表底栏以及搜索页面等所有实例化 `TaskCard` 的场景进行了同步传递，极大地增强了待办事项与其所属清单维度的视觉关联性。
- **简化四象限视图并清理冗余代码**：移除四象限视图设置中残留的自定义象限名称及颜色的编辑功能，简化视图配置；同时从 `SidebarConfigProvider` 及 `SidebarConfig` 模型中彻底清理了已废弃的象限名称和颜色相关的冗余属性及方法，保持代码库的精简和高内聚。
- **重构并统一新建任务弹窗的调用逻辑**：在 `AddTaskBottomSheet` 类中抽象并封装了静态的 `show` 方法，将全项目各处直接通过 `showModalBottomSheet` 手写展示 `AddTaskBottomSheet` 的繁琐代码统一替换为一致的静态方法调用，减少代码冗余并提升可维护性。
- **优化任务详情页与新建对话框的视觉一致性**：调整了任务详情页的主标题字体样式，移存在橙色文字以及下划线装饰，并将字号适配为 20 像素粗体，与新建任务对话框全屏模式下的输入风格保持高度一致，优化了页面跳转切换的视觉体验。
- **优化任务描述 Markdown 编辑工具栏的交互与布局**：修改了任务详情页的描述编辑框（`TaskDescriptionEditor`），使得 Markdown 快捷工具栏只在编辑状态（即获取到输入焦点）时才显示，并将其位置从原先的输入框下方调整到输入框上方，从而提供更紧凑流畅的编辑输入体验。
- **优化任务日程选择器日历组件的今天日期视觉展现**：在使用日历组件构造日期选择面板时，将表示当前“今天”的日期数字背景改为橙色高亮（`Colors.orange`），以提供更清晰的直观视觉对比，方便用户快速定位当前日期。
- **优化底层跨业务模块操作的可靠性与流转性能**：引入领域事件总线（Event Bus），使番茄钟任务关联完成、清单删除与恢复时的任务重新划归 and 找回操作在后台异步流转，解耦核心模块依赖，大幅提升跨模块联动的稳定性和未来操作的可扩展性。
- **重构并解耦撤销（Undo/Redo）历史模块的展示与管理机制**：重构了底层的操作历史引擎，通过定义通用的实体信息展示接口与抽象的数据渲染器注册表，彻底清除了撤销/重做记录详情弹窗与各个具体业务功能模块（任务、习惯、清单等）之间的代码双向强耦合，提高了应用底层核心架构的内聚性和解耦性。
- **内聚与统一日期/时间格式化逻辑**：在 `TimeFormatter` 中集中并重构了“今天”、“明天”、“后天”等相对日期及具体时间的拼接逻辑，移除并替换了日程卡片、搜索页面、日历待办区及新建任务页等各处的重复手写日期格式化，提升了时间展示的一致性。
- **重构清单关联实体的查询机制**：在 `Task` 实体类中内聚封装了 `getChecklist(allChecklists)` 方法，使得卡片 and 列表等各个组件能够直接安全地从清单列表中查询完整的 `ChecklistVO`，消除了先前各组件中重复编写 `firstWhere` 清单匹配逻辑。
- **优化任务优先级颜色与标签的绑定**：在新建任务对话框中完全基于 `TaskPriority` 自身的属性定义进行优先级指示器与颜色样式的动态渲染，杜绝了硬编码映射的重复存在。
- **内聚与重构习惯图标及颜色管理**：在 `IconConstants` 中为所有习惯图标配置了规范化主题颜色，并将习惯列表卡片、打卡详情弹窗及操作撤销历史组件中的手写图标与颜色映射统一内聚为 `IconConstants.getIconByName` 和 `IconConstants.getIconColorByName`，消除了散落的硬编码。
- **消除习惯模块多处手写日期时间格式化**：移除了习惯页面、打卡流水以及归档习惯习惯详情中重复编写的时间和日期字符串拼接代码，统一迁移调用 `TimeFormatter.formatTimeOnly` 和 `DateTimeUtils.formatDate` 进行集中格式化输出。
- **收紧日期选择页面**：保证日期，提醒，重复三个设置可以同时出现在页面当中
- **重构新建任务弹窗结构并解耦状态管理**：将 `AddTaskBottomSheet` 拆分为标准内容组件 `AddTaskStandardContent` 与全屏内容组件 `AddTaskFullScreenContent`，并引入 `AddTaskStateScope` 统一管理和分发新建任务弹窗的状态，实现 UI 与状态逻辑的解耦。
- **全系统引入并应用主题颜色配置（colorTheme）**：在任务卡片、番茄钟、操作撤销历史、清单及侧边栏等多个模块中应用统一的 `colorTheme`，高亮显示活跃的设置项，提升系统视觉一致性。
- **重构任务列表组件并支持看板视图**：提取任务列表组件，将任务列表统一为页签切换的看板视图，提升待办任务看板管理体验。
- **重构侧边栏与日历配置并迁移魔法值为枚举**：将侧边栏配置 `SidebarConfig` 和日历页面配置中的魔术值统一重构为强类型枚举，消除硬编码。
- **优化新建任务弹窗底部工具栏布局**：重构了标准新建任务底栏布局，使工具栏图标尺寸与高度完全对齐，增加水平滚动防护，并将发送按钮（`Icons.send`）完美右对齐。

## 问题修复

- **修复新建任务弹窗 setState 构建期调用异常**：将清单初始匹配与默认赋值逻辑从 `build` 阶段移至 `initState` 阶段，解决因在构建期同步调用 `setState()` 触发的运行时崩溃异常。
- **任务时间/提醒设置不持久化问题修复**
    - 修复了在任务详情页中通过时间与日期选择器修改任务时间后，“提醒”设置无法被正确带入弹窗或回传保存的问题。
    - 修复了 `TaskLifecycleManager` 执行提醒更新操作时遗漏了 `reminderOffsets` 参数，导致多重提醒设定无法持久化同步到 Isar 数据库的问题。
- **习惯打卡类型切换与编译问题修复**
    - 修复了在习惯模型中引入 `HabitType` 枚举后导致的 JSON 反序列化及各个对话框和打卡卡片中的类型不匹配与比较问题。
    - 统一使用 `HabitType` 枚举表示习惯打卡类型，优化了 Isar 的数据库注解 `@Enumerated(EnumType.name)` 以保证数据库向后兼容。
