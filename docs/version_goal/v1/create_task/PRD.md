# 任务创建功能产品需求文档 (PRD)

## 1. 文档概述

### 1.1 背景与目的
为了提升用户记录和管理待办事项的效率，本模块设计为一个**通用的任务创建组件 (`create_task` 模块)**。应用中任何需要创建任务的入口（如待办主页、特定清单页、日历页、四象限页、子任务创建等）都必须统一委托并调用此 `create_task` 模块，以确保用户体验的连贯性和业务逻辑的统一性。

### 1.2 目标用户与使用场景
* **快捷创建**：通过悬浮按钮或快捷入口拉起底部弹窗，聚焦于快速录入任务标题。
* **属性细化**：在创建过程中为任务分配时间、备注、优先级、清单目录、标签、检查点或子任务。
* **跨页面调用**：可以携带初始参数（如预设的清单 ID、预设的优先级或预设的父任务 ID）拉起创建面板。

## 2. 界面结构与布局

任务创建组件根据触发场景和参数配置，支持两种主要的交互形态：
1. **快捷创建底部弹窗 (Quick Add Bottom Sheet)**
2. **完整任务编辑/创建页面 (Full Page Task Editor)**

### 2.1 快捷创建底部弹窗 (Quick Add Bottom Sheet)
* **行为**：弹窗自底部弹出，同时系统键盘自动升起，光标默认聚焦在标题输入框。
* **核心元素**：
  * **文本输入框**：占位符为“想要做点什么？”，用于输入任务名称 (`name`)。
  * **清单选择快捷键**：展示当前预设或默认的清单名称（如“收集箱”），点击可在弹窗内直接切换归属清单。
  * **快捷工具栏 (下置图标组)**：
    * **日期/时间图标** (日历)：点击后直接调用项目现有的自定义日期选择器对话框 [custom_date_picker_dialog.dart](lib/shared/widgets/datetime/custom_date_picker_dialog.dart) 录入截止时间。**不提供任何“今天”、“明天”、“后天”、“周末”等快捷选择按钮。**
    * **提醒/重复图标** (铃铛/重复)：配置提醒开关及通知提前时间，或配置重复规则 (`rrule`)。同样通过通用日期时间选择器中的相应时间/重复设置页完成，不提供快速预设按钮。
    * **优先级图标** (旗帜)：点击快速选择任务优先级等级 (`priority`)。
    * **标签图标** (吊牌)：点击呼出标签选择与新建面板 (`tags`)。
    * **扩展/备注图标** (清单与文本)：点击在弹窗内展开子任务/检查点和描述输入区域。
    * **语音输入图标** (麦克风)：点击进入语音识别录入状态。
  * **保存并发送按钮**：位于输入框右侧的圆形发送/保存按钮。点击后调用保存逻辑，收起弹窗与键盘，并伴随创建成功提示。

### 2.2 展开/扩展属性面板 (More Features Panel)
* **触发方式**：在快捷创建弹窗中，点击快捷工具栏的扩展图标，或者通过键盘折叠/弹窗向上滑动触发。
* **功能组件**：
  * **备注/描述区**：多行文本输入框，对应 `description` 字段。
  * **子任务/检查点区**：
    * 允许用户直接添加多个检查点行（`checkpoints`）。
    * 提供“+ 添加子任务/检查点”按钮。支持点击左侧复选框标记完成度，支持删除和拖拽排序。

### 2.3 完整任务创建/编辑页面 (Full Page Task Editor)
* **触发方式**：点击快捷弹窗的“全屏展开”按钮，或在特定场景（如日历日程新建）中直接由路由导航拉起。
* **布局**：
  * **顶部导航栏**：左侧“取消”/“返回”按钮（放弃更改并确认退出），右侧“保存”按钮。
  * **标题区**：显眼的单行输入框，对应 `name` 字段。
  * **描述区**：宽阔的多行输入框，对应 `description` 字段。
  * **属性列表**：
    * **日期与时间**：点击后唤起项目当前的 `CustomDatePickerDialog` 进行时间选择。
    * **优先级**：点击弹出单选菜单，选择高/中/低/无四种状态。
    * **所属清单**：选择任务归属的清单分类。
    * **标签**：以 Wrap 流式布局展示已绑定的标签，并提供添加标签入口。
    * **检查点列表**：展示并编辑检查点。

## 3. 通用委托设计 (Generic Delegation & Prefill)

为了使 `create_task` 成为全应用通用的任务创建服务，该模块必须支持通过**构造函数/参数列表**接收并填充初始任务状态。

### 3.1 初始状态预填 (Prefill Configuration)
当其他模块委托 `create_task` 模块创建任务时，可传递以下预设参数：
* **`checklistId` (清单 ID)**：例如用户在“工作”清单页面内点击“+”，拉起的创建面板应自动将所属清单设为“工作”（`checklistId` 对应工作清单的 ID）。
* **`priority` (优先级)**：例如用户在“四象限 -> 第一象限（重要且紧急）”中点击添加，拉起的创建面板应默认将优先级设为 `TaskPriority.high`。
* **`startTime` / `endTime` (开始/结束时间)**：例如用户在日历视图的“2026-06-19”网格中长按添加，拉起的面板应默认填充开始日期为该日期。
* **`parentTaskId` (父任务 ID)**：当在某任务详情页中点击“添加子任务”时，应自动将新任务的 `parentTaskId` 设为该任务的 ID，并委托创建。
* **`tags` (预设标签)**：支持传递初始标签列表。

## 4. 功能需求与交互逻辑

### 4.1 日期时间与重复规则设定
* **选用组件**：必须且仅能使用项目当前的 [custom_date_picker_dialog.dart](lib/shared/widgets/datetime/custom_date_picker_dialog.dart) 对话框进行日期选择。
* **时间段与全天**：
  * 任务可被设定为**全天任务** (`isAllDay = true`)，此时只有日期有效，具体时分无效。
  * 任务可包含具体时间或时间段，分别映射至任务的 `startTime` 和 `endTime`。
* **重复规则 (Recurrence)**：
  * 支持自定义重复。生成的重复规则需符合 RFC 5545 标准，并以字符串形式存入 `rrule` 字段。

### 4.2 子任务与检查点交互
* **检查点 (`checkpoints` - List<CheckPoint>)**：
  * 适合轻量级的步骤拆解，不作为独立的 Task 存在。在创建面板中，点击“添加步骤/检查点”可直接输入文本生成。
* **子任务 (`subTaskIds` - List<int>)**：
  * 适合独立的时间管理，有单独的起止时间和优先级。在任务中创建子任务时，应再次委托给 `create_task` 模块进行新任务的生成，并将新生成的任务 `id` 写入父任务的 `subTaskIds` 中，新任务的 `parentTaskId` 指向父任务。

### 4.3 任务保存与撤销逻辑
* **校验逻辑**：标题 (`name`) 不能为空。若为空点击保存，需给用户震动或文本提示。
* **撤销机制**：保存成功后需支持通过全局的 `OperationStackProvider` 压入操作栈。当用户在主界面点击“撤销”时，能够安全地反向撤销此任务的物理/逻辑创建。

## 5. 数据模型映射与技术规格

创建的任务必须严格遵循项目 [task.dart](lib/model/entity/task.dart) 的实体定义，具体映射规则如下：

| UI 字段 / 属性 | `Task` 实体字段 | 数据类型 | 映射与取值逻辑 |
| :--- | :--- | :--- | :--- |
| 任务名称 | `name` | `String` | 标题输入框内容（必填，去首尾空格） |
| 备注 / 描述 | `description` | `String` | 多行备注文本内容，默认为空字符串 `''` |
| 任务状态 | `isDone` | `bool` | 默认创建时为 `false` |
| 全天任务 | `isAllDay` | `bool` | 是否启用全天模式（若启用，则具体时分忽略） |
| 开始时间 | `startTime` | `DateTime?` | 通过 `CustomDatePickerDialog` 选定的日期/时间点 |
| 结束时间 | `endTime` | `DateTime?` | 选定的结束时间段或截止日期 |
| 优先级 | `priority` | `TaskPriority` (Enum) | 对应 `TaskPriority` 枚举值：<br> - 无优先级 -> `none`<br> - 低优先级 -> `low`<br> - 中优先级 -> `medium`<br> - 高优先级 -> `high` |
| 所属清单 | `checklistId` | `int?` | 选定的清单 ID，默认为 `1`（通常是“收集箱/收件箱”） |
| 标签列表 | `tags` | `List<String>` | 绑定的标签字符串数组，默认为空列表 `const []` |
| 重复规则 | `rrule` | `String?` | 重复设置解析出的 RFC 5545 RRule 字符串 |
| 开启通知 | `notificationEnabled`| `bool` | 是否开启提醒通知，默认 `false` |
| 提醒提前时间 | `reminderOffsetMinutes`| `int?` | 提前多少分钟提醒（如 15、30、60 等） |
| 检查点列表 | `checkpoints` | `List<CheckPoint>` | 用户在创建面板中录入的轻量检查步骤列表 |
| 父任务关联 | `parentTaskId` | `int?` | 子任务模式下关联的父任务主键 ID |
| 子任务列表 | `subTaskIds` | `List<int>` | 关联的子任务主键 ID 集合，默认为空列表 `const []` |
