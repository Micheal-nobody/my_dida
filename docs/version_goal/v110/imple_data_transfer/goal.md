# 数据传输功能（导入、导出、删除）

在 v102 版本中，我们新增了数据的备份与恢复功能，以支持本地实体数据的生命周期迁移与传输。

## 预期效果与功能

### 1. 导出数据 (Export Data)
- **入口**：设置 -> 备份与同步 -> 导出数据。
- **二次确认**：点击“导出数据”时弹出二次确认弹窗。
  - 展示即将导出的默认绝对路径预览。
  - 支持直接在文本输入框中编辑并修改该导出路径。
  - **人性化路径选择**：在输入框右侧提供“文件夹”图标按钮，点击后可以调起系统原生的文件保存选择器（`FilePicker.platform.saveFile`）来图形化地定位和选择备份的保存路径。
  - 用户确认后开始导出。
- **数据范围**：
  - Isar 数据库中的所有核心业务实体数据：`Task` (任务)、`Checklist` (清单)、`Habit` (习惯)、`HabitCheckInRecord` (习惯打卡记录)、`TomatoRecord` (番茄钟记录)、`CustomTomato` (自定义番茄钟配置)。
  - **不包含**：页面配置信息（如 `SidebarConfig`, `CalendarPageConfig`）以及历史操作记录数据 (`Operation`, `TaskOperation`)。
- **效果**：生成标准的 JSON 格式文件备份。

### 2. 导入数据 (Import Data)
- **入口**：设置 -> 备份与同步 -> 导入数据。
- **确认与选择**：弹出文件路径输入弹窗（或默认路径预填），确认后从指定的 JSON 备份文件导入数据。
  - **人性化路径选择**：同样在输入框右侧提供“文件夹”图标按钮，点击可调起系统原生文件选择器（`FilePicker.platform.pickFiles`）来图形化选择要导入的 JSON 文件。
- **处理方式**：
  - 执行清空原有核心业务实体数据表。
  - 读取并反序列化 JSON 备份文件，重新写入所有实体，保持实体之间的关联 ID 不变。
  - 自动校验或补齐系统默认必须存在的“收集箱”清单。
  - 导入完成后刷新相关状态管理器并通知 UI 重新加载最新数据。

### 3. 删除数据 (Delete Data)
- **入口**：设置 -> 备份与同步 -> 删除数据。
- **二次确认**：弹出严格的二次确认警告弹窗，提示用户该操作将不可逆地抹去所有业务数据。
- **处理方式**：
  - 清除所有核心业务实体数据（任务、习惯、番茄记录等）。
  - 自动保留或重新初始化默认的“收集箱”清单。
  - 清除完成后刷新状态并提示用户。
