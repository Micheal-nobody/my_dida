# AddTaskDialog 与属性选择优化规划 (Goal)

## 预期效果 (Expected Outcome)

### 1. 属性选择行 (SelectionRow) 选中状态高亮
* 当任务的对应属性（如时间、提醒、重复）被选择生效（非空状态）时，`SelectionRow` 的整行背景应发生高亮变色，而不仅限于改变文本颜色。
* 变色效果采用柔和的浅橙色背景（`Colors.orange.withValues(alpha: 0.1)`）并配有圆角，其左侧图标、行标名称、右侧选中值文本统一自适应高亮为橙色加粗，显著提升视觉选中反馈。

### 2. 提醒选项迁移与多重提醒支持
* **快捷入口迁移**：移除了新建任务对话框（`AddTaskDialog`）底部的“提醒与重复”图标按钮，将“提醒”模块迁移整合至日期与时间选择对话框（`CalendarWidget`）的主列表中，移除其中与“重复”相关的冗余选择，实现统一的时间属性管理页面。
* **多重到期提醒时间**：提醒时间设置由单选升级为多选模式。用户可以在提醒设置行弹出的对话框中勾选并应用多个不同的到期提醒时刻（如任务开始时、提前5分钟、提前15分钟等），从而允许为同一个任务生成多次闹钟通知提醒。
* **时间自动联动提醒**：在日期选择器中，当用户为任务设置了具体的“时间”后，若当前未启用任何提醒，系统将自动添加默认提醒时间为“任务开始时”，以优化快捷交互流程。

### 3. 解决键盘升起时 Dialog 挤压渲染异常
* 修复键盘升起时 `AddTaskDialog` 渲染空间严重压缩、仅显示局部内容的 bug。
* 通过解耦外部 bottom sheet 托起逻辑与内部 Padding 布局，使 `AddTaskDialog` 内部 Container 不再重复受 `viewInsets.bottom` 挤压，在键盘升起时，整个对话框随软键盘整体向上移动，确保输入体验的平滑与内容展示的完整。

## 交付验证 (Verification)
* 运行单元测试（`test/task_service_test.dart` 及 `test/active_reminder_manager_test.dart`）以确保多重提醒在不同平台的计划注册、前台轮询及通知取消的生命周期逻辑均完备且正常。
* 确保在 Android 及其他平台上的静态代码分析中无编译错误与警告。
