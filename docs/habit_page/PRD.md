# 习惯功能模块产品需求文档 (PRD)

## 1. 文档概述

### 1.1 背景与目的
习惯（Habit）是日常时间管理和自我提升的重要维度。与单次性、有时限的“任务”不同，习惯强调长期重复性、坚持打卡与进度量化。为了帮助用户在“滴答清单/My Dida”应用中更好地培养习惯，特设计并规范习惯功能模块。本模块旨在通过直观的主页列表打卡、多维度（日、周、月）的打卡数据分析，以及灵活的管理、归档和筛选系统，帮助用户科学地跟踪与沉淀个人习惯。

### 1.2 目标用户
希望培养良好生活、工作习惯，需要进行日常规律性打卡，并期望通过可视化图表及历史统计进行反思和改进的用户。

---

## 2. 核心功能架构

本模块由以下四大核心部分组成，各子页面与交互入口由主页面顶部应用栏的三个图标按钮承接：
1. **习惯主页面 (对应 `default.jpg`)**：习惯功能的主入口，展示今日打卡列表、打卡进度及提供快速打卡操作。
2. **右上角图标按钮 1 - 打卡数据统计中心 (对应 `data_day_summary.jpg`、`data_week_summary.jpg`、`data_mouth_summary.jpg`)**：提供日、周、月维度的多维度打卡数据图表统计分析。
3. **右上角图标按钮 2 - 习惯管理与归档页 (对应 `habit_manage_page.jpg`、`habit_archived_page.jpg`)**：管理进行中的习惯（支持拖拽排序、归档、修改）及查看已归档的习惯。
4. **右上角图标按钮 3 - 显示范围/过滤弹窗 (对应 `visible_range.jpg`)**：用于对主页面习惯列表进行状态、频次与时间段的组合过滤。

## 3. 详细功能设计

### 3.1 习惯主页面设计 (对应 `default.jpg`)

#### 3.1.1 页面布局与视觉
* **顶部应用栏 (AppBar)**：
  * **标题**：显示“习惯”。
  * **右上角操作栏**：从左到右横向排列三个图标按钮（Icon Button）：
    1. **数据统计图标按钮**（对应 `data_day_summary.jpg`）：点击进入“打卡数据统计中心”；
    2. **习惯管理图标按钮**（对应 `habit_manage_page.jpg`）：点击进入“习惯管理”页面；
    3. **筛选/过滤图标按钮**（对应 `visible_range.jpg`）：点击拉起“显示范围”底部过滤弹窗。
* **习惯卡片列表 (Habit List)**：
  * 展示当前未归档且符合筛选条件的所有今日习惯。
  * **空状态**：若无习惯，展示引导插画（星形空边框）与文案“还没有创建任何习惯，点击右下角 + 按钮创建第一个习惯吧！”。
  * **习惯打卡卡片 (Habit Card)**：
    * **左侧**：展示习惯自定义的 Icon（如跑步、看书、喝水等），Icon 带有圆形背景色。
    * **中部**：
      * 第一行：习惯名称（黑字加粗）。
      * 第二行：打卡频次进度（如“打卡 1/3 次”）和每日提醒时间（如“08:00”）。
    * **右侧**：打卡状态交互区（支持点击快速增加一次打卡进度，并辅以圆形进度条渲染，打卡完成后显示为勾选状态）。
* **底部悬浮按钮 (FAB)**：
  * 右下角常驻带有 “+” 的蓝色圆形按钮，点击后拉起“添加习惯”弹窗（`AddHabitDialog`）。

#### 3.1.2 核心手势与交互逻辑
* **点击卡片**：拉起打卡确认/补卡弹窗（`HabitCheckInDialog`），允许用户手动输入或微调当天的打卡次数。
* **向左滑动 / 菜单操作**：
  * **跳过今天**：点击“跳过今天”并二次确认，将该习惯在今日标记为“已跳过”状态，今日进度不计入失败，但也不增加累计打卡次数，且卡片置灰。
  * **编辑**：拉起编辑习惯弹窗（`EditHabitDialog`）修改习惯属性。
* **向右滑动**：快捷撤销上一次打卡或重置今日打卡进度。

### 3.2 打卡数据统计中心设计

点击 AppBar 右上角第一个图标按钮进入。该页面顶部提供三个页签（Tabs）切换：**日汇总**、**周汇总**、**月汇总**。

#### 3.2.1 日汇总 (对应 `data_day_summary.jpg`)
* **核心内容**：
  * **今日打卡概览**：以大字符重点突出“今日打卡率”（完成打卡习惯数 / 今日应打卡习惯数）与今日总打卡次数。
  * **今日明细列表**：列出今天所有关联的习惯，以及每个习惯具体的打卡次数进度。
  * **打卡时间线**：展示用户在今天具体什么时间点（例如 `08:05 喝水`、`12:30 吃饭`）完成了打卡，呈时间轴排列。

#### 3.2.2 周汇总 (对应 `data_week_summary.jpg`)
* **核心内容**：
  * **周打卡趋势图**：柱状图 (Bar Chart) 或 折线图。横轴为周一至周日（1 - 7），纵轴为每日打卡完成率或打卡次数。
  * **周指标面板**：
    * 本周连续打卡天数。
    * 本周总打卡次数。
  * **习惯本周排行榜**：按本周打卡率从高到低对习惯排序，展示每个习惯“已完成天数/本周应打卡天数”及百分比。

#### 3.2.3 月汇总 (对应 `data_mouth_summary.jpg`)
* **核心内容**：
  * **打卡日历热力图 (Contribution Grid)**：
    * 采用类似于 GitHub 贡献图的方格矩阵或标准的日历网格，按月度天数平铺。
    * 每一个格子的颜色深浅代表当天打卡习惯的数量或打卡总次数（颜色越深表示打卡习惯越多）。
    * 点击日历中的某一天，可在下方展示该天的打卡明细。
  * **月度统计汇总**：
    * 累计打卡次数。
    * 月度打卡率平均值。
    * 历史最长连续打卡天数（习惯流）。

### 3.3 习惯管理与已归档页面设计

点击 AppBar 右上角第二个图标按钮进入。本部分采用页面与子页面组合或多页签（Tabs）结构。

#### 3.3.1 习惯管理页面 (对应 `habit_manage_page.jpg`)
* **核心内容**：
  * 列出所有当前“进行中”的习惯。
  * **拖拽排序**：支持长按习惯项右侧的排序手势按钮进行上下拖拽，以自定义习惯在主页的排列顺序。
  * **快捷操作**：
    * 点击习惯项进入习惯的详情页或直接弹出编辑弹窗。
    * 提供“归档”按钮（点击后习惯将归档，不在主页面展示）。
    * 提供“删除”按钮（删除习惯及所有历史记录，需进行二次弹窗确认）。
  * **跳转入口**：页面顶部或底部提供“已归档习惯”的跳转入口。

#### 3.3.2 已归档习惯页面 (对应 `habit_archived_page.jpg`)
* **核心内容**：
  * 专门展示已被归档的习惯列表。
  * **归档状态规则**：
    * 归档的习惯不再参与日常的主页打卡。
    * 归档的习惯不会触发每日提醒。
    * 归档习惯的历史打卡数据与统计信息仍会被保留在“数据统计中心”中。
  * **操作交互**：
    * **恢复 (Unarchive)**：点击可重新激活该习惯，将其状态变更为进行中，重新放回主页面列表。
    * **彻底删除 (Delete)**：彻底从数据库中抹去该习惯，包含其所有的历史打卡次数和关联统计，不可撤销。

### 3.4 显示范围/过滤弹窗设计 (对应 `visible_range.jpg`)

点击 AppBar 右上角第三个图标按钮触发，自底部弹出半屏工作表（BottomSheet）。

#### 3.4.1 筛选维度
用户可以通过以下三个维度的组合对主页的习惯列表进行过滤：
1. **按打卡状态过滤**：
   * **全部**：显示所有今日习惯。
   * **未完成**：仅显示今天还未达到目标打卡次数的习惯。
   * **已完成**：仅显示今天已经达到或超过打卡次数目标的习惯。
2. **按时段过滤 (根据习惯的提醒时间划分)**：
   * **全天/全部**：显示所有习惯。
   * **早上**：仅显示提醒时间在 `05:00 - 12:00` 之间的习惯。
   * **中午**：仅显示提醒时间在 `12:00 - 18:00` 之间的习惯。
   * **晚上**：仅显示提醒时间在 `18:00 - 24:00` (或次日 `05:00`) 之间的习惯。
3. **按频次/周期过滤**：
   * **全部**：不限频次。
   * **每日**：仅显示每天都需要打卡的习惯。
   * **每周/特定工作日**：仅显示非每日重复（如每周一三五）的习惯。

#### 3.4.2 交互与应用规则
* 弹窗内勾选对应条件后，点击“应用”或“确定”保存设置，主页面列表在 100ms 内即时重构并完成过滤。
* 点击“重置”将所有筛选条件恢复到“全部/未完成”。
* **配置持久化**：筛选过滤条件应保存在本地配置（如 `HabitsPageConfig`）中，确保用户关闭并重新打开应用时过滤状态得以维持。

## 4. 技术与数据逻辑

### 4.1 数据实体扩充
为了满足 PRD 描述的管理、归档和过滤要求，需要对 Isar 实体 `Habit` 进行设计扩展：
* **`isArchived` (bool)**：标记习惯是否已归档。默认值为 `false`。
* **`sortOrder` (int)**：排序权重，用于在习惯管理页面中持久化拖拽排序的结果。
* **`timeSlot` (enum/int)**：通过计算 `remindTime` 的小时数，将其划分到“早上”、“中午”、“晚上”等特定时段类别，用于过滤查询。

### 4.2 打卡流水表设计 (建议)
为实现精细化的打卡时间轴和日/周/月汇总统计，应设计一个独立的打卡记录实体：
```dart
@Collection()
class HabitCheckInRecord {
  Id id = Isar.autoIncrement;
  
  @Index()
  int habitId;           // 关联的习惯 ID
  
  DateTime checkInTime;  // 具体的打卡时间戳
  
  String? note;          // 打卡备注 (可选)
}
```

## 5. UI/UX 体验要求

1. **拖拽顺畅度**：习惯管理页面的列表拖拽排序需伴有流畅的震动反馈与过渡动画，避免出现卡顿或乱序。
2. **过滤即时性**：通过 `visible_range.jpg` 的配置改变过滤条件后，使用 `Provider` 或局部状态驱动，确保页面无闪烁且刷新时间低于 100ms。
3. **视觉一致性**：习惯图标、圆形打卡进度圈以及分类标识的颜色，必须严格采用习惯配置中定义的主题颜色。
