# 日历功能改版与功能增强产品需求文档 (PRD)

## 1. 文档概述

### 1.1 背景与目的
为了提升用户对于时间维度任务管理的效率与视觉体验，现对应用程序的日历页面进行交互改版与功能增强。本次改版将着重于日历视图的切换、无任务时间段的折叠优化、多维度的清单过滤（显示范围选择），以及用户个性化配置的持久化存储。

### 1.2 目标用户
需要根据日程紧凑程度灵活切换日历视图，并希望在日历上过滤无关任务、保留特定清单任务的用户。

---

## 2. 功能范围与边界

### 2.1 包含的功能 (In-Scope)
1. **日历视图模式切换**：支持通过顶部应用栏（AppBar）的切换图标，在不同的日历视图模式（如月视图、周视图、日程视图等）之间进行切换。
2. **时间折叠滑块**：日历网格下方的控制滑块仅用于“折叠/展开无 Task 的时间区域”，从而优化屏幕空间利用率。
3. **“更多”功能菜单**：点击顶部应用栏右侧的“三点”按钮，弹出包含以下选项的菜单：
   * **显示范围**：点击后弹出底部清单过滤选择弹窗。
   * **显示完成任务**：开关/复选框切换，用于控制是否在日历网格和任务列表中展示已完成任务。
4. **清单筛选（显示范围）**：支持级联勾选逻辑（“全部”与“各子清单”联动），可按清单筛选并实时过滤显示在日历网格及底部任务列表中的任务。
5. **偏好配置持久化 (`CalendarPageConfig`)**：通过专门的持久化类保存用户的筛选和视图偏好。

### 2.2 不包含的功能 (Out-Of-Scope)
根据业务规划，`more_features.jpg` 中的以下三个功能本次**不予实现**：
1. **安排任务**
2. **日历订阅**
3. **分享**

## 3. 详细功能说明

### 3.1 主页面布局与交互 (主日历视图)
* **顶部应用栏 (AppBar)**
  * **左侧**：显示当前选中的年份与月份（如“2026年6月”），右侧附带下拉指示箭头。点击可重置/切换至当前月份。
  * **右侧动作按钮**：
    * **搜索按钮**：点击进入任务搜索页面。
    * **视图切换按钮**：点击可弹出视图切换菜单，选择月视图、周视图、日程视图等模式。
    * **更多选项按钮（三点）**：点击弹出下拉菜单。
* **日历网格区 (Calendar Grid)**
  * 根据所选视图展示日期网格。
  * 包含星期栏（“一 二 三 四 五 六 日”）。
  * 日期下方通过彩色的“任务指示点”标注当天有任务。任务指示点需根据任务所属清单的颜色进行着色。
  * 若当天无任务，不显示指示点。若任务指示点对应的任务为“已完成”，则取决于“显示完成任务”的配置来决定是否将其计入指示点渲染。
  * 当前选中的日期以蓝色圆圈高亮显示。
* **时间折叠滑块**
  * 日历网格正下方提供一根带有横线指示器的滑块条。
  * 交互：点击或上下拖拽该滑块，将折叠或展开无 Task 的时间段，减少视觉冗余。
* **底部单日任务列表**
  * 显示当前选中日期的完整日期文本与星期（如“2026年6月19日 星期五”）。
  * 列表渲染该日期下所有满足过滤条件（清单过滤、完成状态过滤）的任务。
  * **已完成任务**：置于列表项中，左侧复选框勾选，文字加上划线并做淡化处理。
  * **未完成任务**：左侧复选框为空，文字正常显示，左侧边缘提供任务优先级颜色条。
  * 交互：
    * 点击任务项进入“任务详情页”。
    * 点击左侧复选框直接切换任务的完成状态（已完成 <-> 未完成）。
* **悬浮操作按钮 (FAB)**
  * 右下角常驻带有 “+” 的圆形悬浮按钮。
  * 交互：点击打开任务快速创建弹窗，默认执行日期为当前选中的日历日期。

### 3.2 更多功能下拉菜单 (More Actions Menu)
* 触发源：AppBar 的“更多选项（三点）”按钮。
* 菜单项列表：
  1. **显示范围**：前置文件夹/清单图标，文案“显示范围”。点击关闭菜单并拉起「显示范围选择弹窗」。
  2. **显示完成任务**：前置复选框/开关图标，文案“显示完成任务”。右侧提供 Switch 或 Checkbox 开关。点击可直接切换状态并应用过滤。

### 3.3 “显示范围”选择弹窗 (Visible Range Sheet)
* 触发源：“更多功能” -> “显示范围”。
* 呈现方式：自下而上滑出的底部工作表（BottomSheet Dialog）。
* 界面组成：
  * 标题：“显示范围”。
  * 选项列表：
    * **“全部”选项**：带有一个复选框或单选按钮。
    * **子清单列表**：罗列用户当前所有的自定义清单（如收集箱、工作、生活、学习等），每个清单左侧显示清单专属颜色的文件夹图标，右侧为独立复选框。
  * 底部动作栏：左侧“取消”按钮，右侧“确定”按钮。
* **级联勾选逻辑规则**：
  * **全部同步**：当用户勾选“全部”时，下方所有的子清单应自动全部变为被勾选状态。
  * **反向失效**：在“全部”已勾选的状态下，若用户手动取消勾选任意一个子清单，则顶部的“全部”选项自动变为未勾选状态。
  * **正向点亮**：当用户手动将所有子清单全部勾选时，顶部的“全部”选项自动变为已勾选状态。
* **按钮动作**：
  * **确定**：保存当前的清单筛选组合至 `CalendarPageConfig` 中，触发日历页面的数据刷新，并关闭底部弹窗。
  * **取消**：丢弃本次弹窗内的所有修改，关闭弹窗，日历页面保持原状。

## 4. 技术与数据逻辑

### 4.1 配置管理实体 `CalendarPageConfig`
新建一个独立的配置实体或持久化模型，用于存放日历页面的个性化显示配置：
```dart
class CalendarPageConfig {
  bool showCompletedTasks;        // 是否显示已完成任务
  CalendarVisibleMode visibleMode; // 清单筛选模式：全部显示 / 自定义清单过滤
  List<String> visibleChecklistIds; // 自定义清单过滤下，可见的清单 ID 列表
  CalendarViewMode viewMode;       // 视图模式：月 / 周 / 日程等
  bool isTimeFolded;               // 是否折叠无任务的时间段
}
```

### 4.2 数据过滤逻辑
在加载日历网格的任务标记点及底部单日任务列表时，应用以下级联过滤算法：
1. **清单过滤**：
   * 若 `visibleMode` 为 `all`（全部），加载所有清单的任务。
   * 若 `visibleMode` 为 `customChecklists`（自定义），则只加载 `checklistId` 存在于 `visibleChecklistIds` 中的任务。
2. **完成状态过滤**：
   * 若 `showCompletedTasks` 为 `true`，加载已完成和未完成的任务。
   * 若 `showCompletedTasks` 为 `false`，仅加载未完成的任务。

### 4.3 持久化存储规范
* 所有的配置变更必须实时或在关闭页面前写入本地存储（Isar 数据库对应的配置表，或者 Shared Preferences 存储中）。
* 每次进入日历页面时，应优先从持久化存储中读取 `CalendarPageConfig` 以恢复用户上次离开时的视图和筛选状态。

## 5. UI/UX 体验要求
1. **即时刷新**：在更改“显示范围”或“显示完成任务”设置后，页面应通过 `Provider` 驱动在 100ms 内完成过滤数据的重新渲染，无明显白屏或卡顿。
2. **过渡动画**：月视图与周视图之间的折叠切换、无任务时间段的折叠切换，应伴有平滑的垂直折叠过渡动画。
3. **色彩一致性**：日历指示点的颜色、清单图标的颜色、任务条的左侧标志线颜色，必须与清单定义的 `color` 保持严格一致。
