# 任务页面侧边栏产品需求文档 (PRD)

## 1. 项目概述

### 1.1 背景与目标
侧边栏（Sidebar/Drawer）是应用内全局导航与任务分类管理的核心枢纽。用户需要通过侧边栏快速切换不同的任务视图（如今日任务、近七天任务、自定义清单、标签等），进行全局搜索，并进入系统设置。
本项目旨在规范并实现任务侧边栏的界面展现、个性化定制（包括智能清单显隐控制、侧边栏模块开关及底栏快捷键定制），提升任务检索和切换的效率。

### 1.2 目标用户与使用场景
* **高效时间管理者**：快速切换到“今天”或“最近七天”查看紧迫任务。
* **多项目协作者/个人分类者**：根据自定义的“清单”（如工作、学习、生活）和“标签”分类查看并组织任务。
* **极简主义用户**：通过侧边栏设置隐藏不常用的模块（如搜索框、标签、特定智能清单），只保留核心视图。

## 2. 界面设计与交互细则

侧边栏整体采用经典的抽屉式设计（Drawer），可从屏幕左侧边缘右滑或点击主页左上角菜单按钮唤出。

### 2.1 侧边栏默认视图 (`default.jpg`)

默认视图自上而下包含以下六个核心区块：

#### 2.1.1 用户信息区 (Profile Section)
* **元素组成**：
  * 用户圆形头像。
  * 用户昵称（如：`Michel-nobody`）。
  * 尊贵身份标识/等级（如金色金牌/皇冠图标，标示会员或成就状态）。
  * 用户账号邮箱（如：`2201389816@qq.com`）。
* **交互**：点击头像或昵称区域，直接跳转至“个人中心”或“设置”界面。

#### 2.1.2 搜索区 (Search Section)
* **元素组成**：一个通栏搜索框，带有放大镜图标和提示占位文本“搜索”。
* **交互**：点击搜索框，拉起全局搜索界面，允许输入关键字检索任务内容。

#### 2.1.3 智能清单导航区 (Smart Lists Section)
* 显示处于激活状态的内置智能清单，默认包含：
  * **今天**：蓝色日历图标（图标内展示今日日期数字），右侧显示今日待办任务的未完成数量。
  * **最近七天**：紫色多日历堆叠图标，右侧显示未来七天内的待办任务数量。
  * **收集箱**：蓝色收纳箱图标，用于承载未分类的待办事项，右侧显示数量。
* 样式与分割线：智能清单区下方有一条水平分割线，与自定义清单区隔开。

#### 2.1.4 自定义清单区 (Custom Lists Section)
* **元素组成**：
  * **清单标题**：左侧为展开/折叠箭头，中间为“清单”二字，右侧为“+”号按钮（快捷“创建新清单”）。
  * **清单列表**：展示用户创建的清单（如“工作”、“学习”、“生活”）。每个清单左侧有彩色的圆形点（标识清单颜色或专属图标），右侧显示该清单下的待办任务数量。
* **交互**：
  * 点击清单名称切换至对应的任务列表视图。
  * 点击“清单”右侧的“+”可以弹窗快速创建清单。

#### 2.1.5 标签区 (Tags Section)
* **元素组成**：
  * **标签标题**：左侧为展开/折叠箭头，中间为“标签”二字，右侧为“+”号按钮（快捷“创建新标签”）。
  * **状态展示**：若无标签，展示灰色提示文本“暂无标签”。若有标签，以列表形式展示各个标签及其颜色标识。

#### 2.1.6 底部固定工具栏 (Bottom Toolbar Section)
* **元素组成**：固定悬浮在侧边栏最底部，包含：
  * **左侧按钮**（如“+”或清单添加图标）：功能为“新增自定义清单”，点击弹出创建新清单的交互界面。
  * **右侧按钮**（如齿轮设置图标）：功能为“全局设置与智能清单配置”的入口，点击跳转至系统设置界面。

### 2.2 全局设置与智能清单配置 (`setting1.jpg`, `setting2.jpg`)

#### 2.2.1 智能清单入口
在“设置”页面 (`setting1.jpg`) -> “任务管理”分组下，点击**“智能清单”**可进入智能清单显隐配置界面。

#### 2.2.2 显隐配置页面 (`setting2.jpg`)
允许用户独立配置每一个系统级智能清单在侧边栏的显示策略。
* **清单范围**：包括“今天”、“明天”、“最近七天”、“收集箱”、“所有”、“已完成”、“垃圾桶”。
* **显示策略选项**：
  * **显示 (Show)**：无论是否有任务，始终在侧边栏显示。
  * **隐藏 (Hide)**：始终在侧边栏隐藏该清单。
  * **自动 (Auto/Show if active)**：仅当该清单下有未完成任务（或存在任务）时才在侧边栏显示，无任务时自动隐藏。
* **默认项状态**（依据截图）：
  * “今天” -> 显示
  * “明天” -> 自动
  * “最近七天” -> 显示
  * “收集箱” -> 显示
  * “所有” -> 隐藏
  * “已完成” -> 隐藏
  * “垃圾桶” -> 隐藏

---

### 2.3 侧边栏定制配置 (`setting3.jpg`)

在“设置”页面或侧边栏底栏可进入“侧边栏”专属定制界面，用于控制侧边栏顶层大模块的可见性及底栏功能：

* **模块可见性开关**：
  * **头像与用户信息**（开关）：关闭后隐藏顶部用户头像、昵称及邮箱区域，整体导航区上移。
  * **搜索**（开关）：关闭后隐藏搜索输入框。
  * **智能清单**（开关）：控制是否整体显示智能清单模块。
  * **清单**（开关）：控制是否显示自定义清单模块。
  * **标签**（开关）：控制是否显示标签模块。
  * **过滤器**（开关）：控制是否在侧边栏展示自定义过滤器模块。
* **排序与管理**：
  * 清单排序支持“手动拖拽”或“按字母/创建时间”自动排序。
* **底栏自定义配置**：
  * 用户可以勾选或拖拽排序底部工具栏的快捷按钮（如隐藏统计按钮，只保留设置按钮，或加入日历快捷方式）。

## 3. 功能需求与业务逻辑

### 3.1 计数统计规则
* **待办任务数**：侧边栏各清单右侧显示的数字代表该清单下**未完成 (Active/Uncompleted) 且未被软删除**的待办任务总数。
* **过期任务**：已过期但未完成的任务，应默认计入“今天”或对应清单的未完成计数中，并根据设置项进行逾期高亮。
* **收集箱计数**：仅统计放入“收集箱”中且未标记完成的任务。

### 3.2 智能清单筛选器逻辑 (Filter Rules)
* **今天**：
  * `task.dueDate == 今日` 且 `task.isCompleted == false` 且 `task.isDeleted == false`
* **明天**：
  * `task.dueDate == 明日` 且 `task.isCompleted == false` 且 `task.isDeleted == false`
* **最近七天**：
  * `今日 <= task.dueDate <= 今日+6天` 且 `task.isCompleted == false` 且 `task.isDeleted == false`
* **所有**：
  * `task.isCompleted == false` 且 `task.isDeleted == false`
* **已完成**：
  * `task.isCompleted == true` 且 `task.isDeleted == false`
* **垃圾桶**：
  * `task.isDeleted == true`（从删除起保存一定天数，如 30 天后物理删除）

### 3.3 状态同步与持久化
* 侧边栏的每一项定制设置（如各智能清单的“显示/隐藏/自动”状态，模块开关状态，底栏排列等）均需持久化保存在本地数据库（如 Isar 中的本地配置表/Key-Value Settings）。
* 数据同步点击后，调用同步服务，同步成功后刷新侧边栏的任务数字计数。

## 4. 技术设计参考

### 4.1 数据模型 (Data Model)
在本地 Isar 数据库中需要引入或扩展 `SidebarConfig` 实体，以记录侧边栏组件的偏好：

```dart
@collection
class SidebarConfig {
  Id id = Isar.autoIncrement;

  // 模块开关
  bool showProfile = true;
  bool showSearch = true;
  bool showSmartLists = true;
  bool showCustomLists = true;
  bool showTags = true;
  bool showFilters = false;

  // 智能清单显示配置 (0: 隐藏, 1: 显示, 2: 自动)
  int todayShowOption = 1; 
  int tomorrowShowOption = 2;
  int nextSevenDaysShowOption = 1;
  int inboxShowOption = 1;
  int allShowOption = 0;
  int completedShowOption = 0;
  int trashShowOption = 0;

  // 底栏配置
  List<String> bottomBarItems = ['settings', 'statistics'];
}
```

### 4.2 状态管理与刷新
* 侧边栏的状态更新应由 `TaskProvider` 或专门的 `SidebarProvider` 驱动。
* 当任务状态改变（如标记完成、修改截止日期、新建任务）时，对应的 Provider 需要触发重建，重新计算各清单的数字，并判断设置了“自动显示”的智能清单是否需要动态呈现。

## 5. 交互与体验要求 (UX/UI Requirements)

* **侧边栏滑出阻尼**：右滑拉出侧边栏应带有适当的物理阻尼感，且滑出动画时间控制在 250ms 内，确保流畅无卡顿。
* **数字缩写**：若任务数量超过 999，右侧气泡或数字显示为 `999+`，避免撑破容器。
* **高亮反馈**：当前处于选中激活状态的清单/智能清单，在侧边栏中应有明显的底色高亮（如浅蓝色或主题色半透明背景），图标及文字颜色也应同步切换为主色调。
* **空白状态处理**：当标签或清单折叠或为空时，提供友好的文字引导（如“暂无标签，点击 + 创建”）。
